Sistema de Controle de Acesso Escolar - Especificação de Requisitos 
Requisitos Funcionais (RF) 
Dashboard Administrativo 
Painel central com visão geral do sistema 
Status das catracas em tempo real (online/offline) 
Resumo de acessos recentes (últimas 24 horas) 
Indicadores de performance (total de acessos, tentativas bloqueadas) 
Alertas e notificações importantes 
Gráficos de fluxo de entrada/saída por horário 
CRUD para: 
Alunos: matrícula, nome completo, foto, status (ativo/inativo), turma, 
horários permitidos 
Turmas: código, nome, horários de entrada/saída 
Usuários Administrativos: nome, email, nível de acesso, permissões 
específicas, histórico de login 
Catracas: identificação, localização, modelo ControlID, status, última 
sincronização 
Autorizações Temporárias: aluno, período de validade, motivo, 
responsável pela aprovação 
Registro de Logs: timestamp, usuário, ação realizada, dados alterados, IP de origem

Configuração de feriados e dias letivos especiais 
Sistema de exceções por aluno (atendimento médico, reforço escolar)

Sincronização automática com catracas ControlID 
Backup e restauração de configurações das catracas 
Interação com Usuários

Interface de consulta para secretaria escolar 
Visualização de relatórios por responsáveis (mediante autorização) 
Sistema de notificações por email para eventos importantes 
Chat interno para comunicação entre administradores 
Monitoramento e Auditoria 
Registro completo de todos os acessos (entrada/saída) 
Log de tentativas de acesso negadas com motivo 
Histórico de alterações no sistema com rastreabilidade 
Monitoramento de performance das catracas 
Detecção de padrões anômalos de acesso

Regras de Negócio (RN) 
Gestão Integrada do Controle de Acesso 
RN001: O sistema deve atender administradores escolares, secretaria, 
direção e TI de forma centralizada 
RN002: Apenas usuários autenticados e autorizados podem acessar o 
sistema administrativo 
RN003: Diferentes níveis de permissão: Visualizador, Operador, 
Administrador, Super Administrador 
RN004: Todas as ações críticas devem ser registradas em log de auditoria 
Controle de Acesso e Horários 
RN005: Aluno só pode acessar durante horários definidos para sua turma RN006: Autorizações temporárias têm prioridade sobre horários de turma 
RN007: Bloqueios manuais impedem acesso independente de outras 
regras 
        
           RN008: Aluno inativo não pode acessar o sistema 


RN009: Máximo de 3 tentativas de acesso em sequência (anti-tailgating) 
Sistema de Controle de Acesso Escolar  Requisitos 4








Segurança e Privacidade 
RN010: Dados biométricos ficam exclusivamente nas catracas (edge 
computing) 
RN011: Consentimento obrigatório para coleta e uso de biometria 
RN012: Retenção de logs por no máximo 5 anos 
RN013: Direito ao esquecimento deve ser implementado conforme LGPD RN014: Acesso aos dados pessoais limitado por necessidade e função 
Operação e Contingência 
RN015: Catracas devem funcionar offline (modo standalone) por até 72 
horas 
RN016: Sistema deve alertar sobre falhas de comunicação em até 5 
minutos 
RN017: Backup automático não pode interferir na operação normal 
RN018: Sincronização deve ocorrer fora dos horários de pico de acesso RN019: Plano de contingência deve estar sempre atualizado 
Escalabilidade e Crescimento 
RN020: Arquitetura deve suportar expansão para múltiplas unidades 
escolares 
RN021: Sistema deve comportar crescimento de até 100% da base atual sem reestruturação 
RN022: Novas catracas podem ser adicionadas sem interrupção do serviço 
RN023: Integração com novos sistemas deve ser possível via API 
padronizada

Critérios de Aceitação 
Funcionais
✅ 100% dos alunos cadastrados conseguem acessar nos horários corretos ✅ 0 falsos positivos em acessos não autorizados 
✅ Relatórios gerados em menos de 10 segundos 
✅ Sincronização completa em menos de 2 minutos 
Não Funcionais 
✅ Zero perda de dados em 12 meses 
✅ Conformidade LGPD 100% auditada 
Usabilidade 
✅ Usuário básico operando sistema em até 1 hora de treinamento 
✅ Taxa de erro de usuário ≤ 2% 
✅ Satisfação do usuário ≥ 8/10 
✅ Suporte técnico resolvendo 90% dos chamados em até 4 horas
